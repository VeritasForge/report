Confluence 주간 데일리 페이지들을 읽고 주간 요약 보고서를 생성합니다.

## 입력 파라미터
- `$ARGUMENTS`: `SPACE_KEY "MENTION_USERS"` 형식
  - SPACE_KEY: Confluence 스페이스 키 (기본값: MAI)
  - MENTION_USERS: 지연/보류 시 멘션할 사용자 (선택)
  - 예: `MAI "@홍길동 @김철수"` 또는 `MAI`

## 실행 단계

### 1. 지난주 날짜 범위 계산 및 Confluence 페이지 읽기

**오늘 날짜를 기준으로 지난주 월요일~금요일 범위를 계산합니다:**
1. 오늘 날짜 확인 (예: 2026-02-02, 월요일)
2. 지난주 월요일 날짜 계산 (오늘 기준 이전 주 월요일)
3. 지난주 금요일 날짜 계산 (지난주 월요일 + 4일)
4. 페이지 제목 형식으로 변환: `YYYY.MM.DD ~ DD`
   - 예: 월요일이 2026-01-26, 금요일이 2026-01-30이면 → `2026.01.26 ~ 30`

**`mcp__mcp-atlassian__confluence_get_page` 도구를 사용하여 페이지를 검색합니다:**
- `space_key`: $ARGUMENTS의 첫 번째 값 또는 기본값 `MAI`
- `title`: 위에서 계산한 페이지 제목 (예: "2026.01.26 ~ 30")
- `include_metadata`: true
- `convert_to_markdown`: true

### 2. 지난주 전체 작업 내용 추출
페이지 내용에서 **지난주 월요일부터 금요일까지** 각 팀원별 작업 내용을 모두 추출합니다:
- Done (완료)
- Doing (진행 중)
- ToDo (예정)

**주의**: Daily report와 달리, 한 주 전체(5일분)의 작업 내용을 수집합니다.

### 3. sequential-thinking MCP를 활용한 주간 분석
`mcp__sequential-thinking__sequentialthinking` 도구를 사용하여 다음을 분석합니다:
- 제품별 분류 (VC 2.2.0, ER 1.0.0 등)
- 주간 완료된 작업 요약
- 주간에 걸쳐 진행 중인 항목 식별
- 일정 지연 여부 판단
- 담당자별 작업 매핑
- 리스크/이슈 식별

### 4. 주간 요약 보고서 형식으로 정리 (슬랙 호환)
다음 형식으로 최종 보고서를 작성합니다.
**중요: 슬랙에 복붙할 수 있도록 마크다운 테이블 사용 금지. 리스트 형태로만 작성.**

```
*📊 주간 요약 ([YY.MM.DD ~ MM.DD])*

*[제품명]*
• ✅ [완료된 주요 작업] - [담당자]
• 🔄 [진행 중 작업] (진행률 또는 예상 완료일) - [담당자]
• ⚠️ [지연 항목] - [담당자] (사유: [사유])
───────────────────
*[제품명]*
• ✅ [완료된 주요 작업] - [담당자]
• 🔄 [진행 중 작업] - [담당자]
───────────────────
*📌 기타*
• 🏖️ [휴가/반차] - [담당자]
───────────────────
⚠️ 지연/보류 항목이 있어 공유드립니다.
[MENTION_USERS]
```

**참고**:
- 지연(⚠️, 🚨) 또는 보류/대기(⏸️) 항목이 없는 경우 마지막 멘션 섹션은 생략합니다.
- `[MENTION_USERS]`는 $ARGUMENTS의 두 번째 값으로 대체합니다. (예: "@홍길동 @김철수")
- 멘션 사용자가 제공되지 않은 경우 멘션 섹션 전체를 생략합니다.

## 포맷 규칙

### 상태 이모티콘
- ✅ 완료
- 🔄 진행 중 (주간에 걸쳐 진행 중인 항목)
- ⚠️ 소폭 지연
- 🚨 크리티컬 지연
- ⏸️ 보류/대기
- 🏖️ 휴가/반차

### 제품명 표기
- 제품명은 반드시 *[제품명]* 형식으로 표기 (Bold + 대괄호)
- 예: *[PRD1 릴리즈]*, *[PRD2 데모]*, *[ETC]*

### 구분선
- 섹션 구분은 `───────────────────` 사용

### 날짜 범위 표기
- 주간 요약 헤더에 지난주 날짜 범위 표시: `[YY.MM.DD ~ MM.DD]`
- 예: `*📊 주간 요약 ([26.01.26 ~ 01.30])*`

## 주의사항
- C-Level이 볼 문서이므로 **주간 핵심 성과와 진행 상황**에 집중
- 일별 세부사항이 아닌 **주간 단위로 통합 요약**
- 동일 작업이 여러 날에 걸쳐 있으면 하나로 통합하여 표시
- JIRA 티켓 번호는 생략하고 작업 내용으로 추상화
- 기술적 세부사항보다는 비즈니스 임팩트 중심으로 기술
- **슬랙 호환 필수**: 마크다운 테이블(|---|) 사용 금지, 리스트(•) 형태로만 작성
- 슬랙 굵은 글씨는 *텍스트*, 기울임은 _텍스트_ 형식 사용
- **지연/보류 알림**: ⚠️, 🚨, ⏸️ 상태 항목이 있고 멘션 사용자가 제공된 경우, 리포트 하단에 해당 사용자 멘션 추가
